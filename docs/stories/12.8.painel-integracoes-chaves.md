## Story

Como **usuario que administra sua organizacao**, eu quero configurar minhas proprias chaves de API (VAPI, Twilio, ElevenLabs) de forma simples e visual, para que meu agente de voz funcione com minhas contas sem depender de suporte tecnico.

---

## Problem

Hoje todas as API keys (VAPI, Twilio, OpenAI, Google) estao hardcoded no `.env.local` do servidor, compartilhadas entre todos os orgs. Isso significa que:
1. Cada cliente nao pode ter suas proprias credenciais
2. O dono do sistema paga por todos os usos
3. Se uma key for revogada, afeta todos os clientes
4. Nao ha como o usuario saber se suas integracoes estao funcionando

**Mascaramento de keys na API de leitura:**
```typescript
function maskKey(key: string): string {
  if (!key || key.length < 8) return '••••••••'
  return '••••••••' + key.slice(-5)
}

// GET /api/call-routing/integrations
export async function GET(req) {
  const config = await getCallRoutingConfig(orgId)
  const integrations = config?.integrations || {}

  // Mascarar todas as keys antes de retornar
  return NextResponse.json({
    vapi: integrations.vapi ? {
      ...integrations.vapi,
      apiKey: maskKey(integrations.vapi.apiKey),
    } : null,
    twilio: integrations.twilio ? {
      ...integrations.twilio,
      accountSid: maskKey(integrations.twilio.accountSid),
      authToken: maskKey(integrations.twilio.authToken),
    } : null,
    // ...
  })
}
```

**Refatoracao de `callRouting.ts`:**
```typescript
// ANTES (linha 22):
const VAPI_API_KEY = process.env.VAPI_API_KEY

// DEPOIS:
function getVapiApiKey(config?: CallRoutingConfig | null): string {
  return config?.integrations?.vapi?.apiKey
    || process.env.VAPI_API_KEY
    || ''
}

// Todas as funcoes que usam VAPI_API_KEY passam a chamar getVapiApiKey(config)
```
