# Story 24.2 — Probabilidade de Fechamento Integrada ao Progresso do Funil

**Status:** InReview
**Epic:** 24 — Refinamentos de UX no Funil e Ligações
**Points:** 3
**Priority:** High
**Dependencies:** Story 24.1 (header do painel reorganizado com dropdown de funil)

---

## Story

Como usuário do CRM, quero que a probabilidade de fechamento esteja integrada visualmente ao progresso do funil (usando o campo existente de prospecção/progresso), em vez de ser um campo separado, para poder ajustar manualmente o % de probabilidade e ver o cálculo de faturamento esperado de forma contextualizada.

---

## Problem

Atualmente a probabilidade de fechamento é exibida como um bloco amber isolado no painel lateral (`closingProbability` — linhas 5694-5732). Isso causa:
1. Desconexão visual entre a probabilidade e o progresso no funil
2. O campo parece "solto" e não contextualizado
3. O usuário não entende a relação entre etapa, probabilidade e valor esperado

A probabilidade deve estar integrada ao contexto do funil/etapa, junto com os badges no header, tornando claro que é o **progresso no funil** que pode ser ajustado manualmente.

---

## Acceptance Criteria

### AC1: Probabilidade exibida junto ao contexto de etapa no header
- **Given** o painel de detalhe de um cliente aberto
- **When** o usuário olha para o header (onde ficam os badges de funil e etapa)
- **Then** vê o % de probabilidade exibido de forma integrada (junto ou abaixo dos badges)
- **And** o valor exibido é: `closingProbability` do cliente se definido, senão `probability` da etapa
- **And** é claro visualmente que o % está ligado ao progresso no funil

### AC2: Edição inline da probabilidade
- **Given** a probabilidade exibida no contexto do header
- **When** o usuário clica no valor de probabilidade
- **Then** transforma em input editável (inline)
- **And** aceita valores de 0 a 100
- **And** salva no `onBlur` (não onChange) para minimizar writes no Firestore
- **And** mostra indicação visual de que foi personalizado (ex: "personalizado" badge)

### AC3: Valor esperado de faturamento calculado
- **Given** o cliente tem `dealValue` definido e probabilidade visível
- **When** o painel é exibido
- **Then** mostra o valor esperado calculado: `dealValue × probabilidade / 100`
- **And** o cálculo usa `getClientProbability(client, stage)` como base
- **And** exibido em formato currency PT-BR (R$ X.XXX)

### AC4: Remover bloco amber separado
- **Given** o novo layout integrado implementado
- **When** o painel de detalhe é exibido
- **Then** o bloco amber separado "Probabilidade de Fechamento" (linhas 5694-5732) NÃO existe mais
- **And** toda a funcionalidade está no novo componente integrado no header

### AC5: Reset ao mudar de etapa preservado
- **Given** a probabilidade integrada ao header
- **When** o cliente muda de etapa (drag & drop, dropdown de etapa, ou mudança de funil)
- **Then** o `closingProbability` é resetado (deleteField)
- **And** volta a exibir a probabilidade default da nova etapa
- **And** o valor esperado é recalculado

---

## Scope

### IN Scope
- Exibição integrada de probabilidade no contexto do header (funil/etapa)
- Input inline editável de probabilidade (click-to-edit)
- Exibição de valor esperado (dealValue × probabilidade)
- Remoção do bloco amber antigo
- Manter lógica de `handleUpdateProbability()` e `getClientProbability()`

### OUT of Scope
- Alteração nos KPIs do kanban (já usam `getClientProbability`)
- Edição de probabilidade em massa
- Histórico de mudanças de probabilidade
- Slider visual (manter input numérico simples)

## Dev Notes

### Layout sugerido no header (após Story 24.1)
```
[Avatar] Nome do Contato
         Empresa
         [🔵 Funil Comercial ▾] [🟢 Qualificação ▾]
         📊 65% · Valor esperado: R$ 32.500
              ↑ clicável para editar
```

Ou alternativamente, com um pequeno bloco compacto abaixo dos badges:
```
[🔵 Funil Comercial ▾] [🟢 Qualificação ▾]
[📊 65% ✏️] → R$ 32.500 esperado
```

### Componente de probabilidade inline
- Estado normal: exibe `XX%` como texto com ícone de edição sutil
- Ao clicar: transforma em `<input type="number" min="0" max="100" />`
- onBlur: salva e volta para modo exibição
- Se personalizado: mostra badge pequeno "(personalizado)" ou ícone diferente
- Se default da etapa: mostra mais discreto

### Valor esperado
```typescript
const probability = getClientProbability(selectedClient, currentStage)
const expectedValue = selectedClient.dealValue
  ? (selectedClient.dealValue * probability) / 100
  : null
```
