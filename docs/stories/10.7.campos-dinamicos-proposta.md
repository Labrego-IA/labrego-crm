# Story 10.7 — Campos Dinamicos na Proposta (Admin)

**Status:** Ready for Review
**Epic:** 10 — Revisao de Propostas Comerciais
**Priority:** High
**Complexity:** High (13-21 points)
**Depends On:** Story 10.6

---

## Story

Como **admin de uma organizacao**, eu quero poder adicionar campos personalizados ao formulario de proposta (definindo nome, tipo e posicao), para que cada organizacao possa adaptar a proposta ao seu modelo de negocio sem depender de desenvolvimento.

---

## Problem

O formulario de proposta tem campos fixos no codigo. Se uma organizacao precisa de um campo extra (ex: "Prazo de entrega", "Garantia", "SLA", "Responsavel tecnico"), nao ha como adicionar sem alterar o codigo-fonte. Num SaaS multi-tenant, cada org tem necessidades diferentes.

## Dev Notes

**Tipo sugerido:**
```typescript
interface ProposalCustomField {
  id: string
  key: string          // identificador unico (slug)
  label: string        // label exibido no form
  type: 'text' | 'number' | 'textarea' | 'select' | 'checkbox'
  required: boolean
  position: 'after_context' | 'after_products' | 'after_observations' | 'after_payment'
  options?: string[]   // para tipo select
  order: number        // ordem entre campos na mesma posicao
}
```

**Firestore:**
- Config: `organizations/{orgId}/settings/proposalCustomFields` → `{ fields: ProposalCustomField[] }`
- Dados: Cada proposta em `proposals` recebe campo `customFields: Record<string, any>`

**Renderizacao no formulario:**
- Agrupar campos por `position`
- Renderizar entre as secoes fixas na posicao definida
- Usar `register()` do react-hook-form com path `customFields.{key}`

**Renderizacao no PDF:**
- Nova secao "Informacoes Adicionais" ou inline entre secoes
- Checkbox: mostrar como "Sim/Nao"
- Select: mostrar valor selecionado
- Textarea: respeitar quebras de linha

**CUIDADO:**
- Campos podem ser adicionados/removidos pelo admin a qualquer momento
- Propostas existentes podem nao ter campos que foram adicionados depois
- Formulario de edicao deve lidar com campos faltantes graciosamente
