# Story 21.2 — Probabilidade Individual + Mover entre Funis

**Status:** Ready
**Epic:** 21 — Pipeline de Vendas & KPIs de Funil
**Points:** 5
**Priority:** High
**Dependencies:** Story 21.1 (KPIs usam probabilidade)

---

## Story

Como usuário do CRM, quero poder editar a probabilidade de fechamento individualmente por contato (sobrescrevendo o default da etapa) e poder mover um contato para outro funil diretamente no painel de detalhe, para ter flexibilidade na gestão do pipeline sem depender apenas dos parâmetros da etapa.

---

## Acceptance Criteria

### AC1: Probabilidade individual editável no detalhe do cliente
- **Given** o painel de detalhe de um cliente aberto no funil
- **When** o usuário visualiza a seção de progresso/probabilidade
- **Then** vê um campo numérico editável com a probabilidade atual
- **And** o valor default é a probabilidade da etapa atual (stage.probability)
- **And** se o usuário alterar, salva em `closingProbability` no documento do cliente no Firestore
- **And** o campo usa padrão onBlur (não onChange) para evitar writes excessivos
- **And** aceita valores de 0 a 100 (com validação)
- **And** exibe label "Probabilidade de Fechamento" com ícone de porcentagem

### AC2: Prioridade de probabilidade nos cálculos
- **Given** um cliente com `closingProbability` definido
- **When** os KPIs de valor esperado são calculados (Story 21.1)
- **Then** usa `client.closingProbability` se existir, senão usa `stage.probability`
- **And** a função helper `getClientProbability(client, stage)` centraliza essa lógica

### AC3: Mover cliente para outro funil no detalhe
- **Given** o painel de detalhe de um cliente aberto
- **When** o usuário clica em "Mover para outro funil"
- **Then** abre um dropdown/modal com a lista de funis disponíveis da organização
- **And** ao selecionar um funil, mostra as etapas daquele funil para o usuário escolher
- **And** ao confirmar, atualiza no Firestore:
  - `funnelId` → novo funil selecionado
  - `funnelStage` → etapa selecionada
  - `funnelStageUpdatedAt` → timestamp atual
  - Limpa campos de cadência (`currentCadenceStepId`, `lastCadenceActionAt`, `lastCadenceStepResponded`)
  - Reseta `closingProbability` (para assumir probability da nova etapa)
- **And** adiciona log no subcollection `clients/{id}/logs` com texto "Movido para funil X → etapa Y"
- **And** exibe toast de sucesso
- **And** o cliente desaparece da view atual do kanban (pois mudou de funil)

### AC4: Resetar probabilidade individual
- **Given** um cliente com `closingProbability` personalizado
- **When** é movido de etapa (drag & drop ou mudança manual)
- **Then** o campo `closingProbability` é removido/resetado
- **And** volta a usar a probabilidade default da nova etapa

---

## Scope

### IN Scope
- Campo `closingProbability` no modelo do cliente (Firestore)
- Input editável de probabilidade no detalhe do cliente (onBlur, 0-100)
- Helper `getClientProbability(client, stage)` para uso em cálculos
- Dropdown/modal de "Mover para outro funil" com seleção de funil + etapa
- Atualização Firestore ao mover (funnelId, funnelStage, limpar cadência)
- Log de movimentação
- Reset de closingProbability ao mudar de etapa

### OUT of Scope
- Edição de probabilidade em massa
- Página de projeção de vendas (Story 21.3)
- Histórico de movimentações entre funis
