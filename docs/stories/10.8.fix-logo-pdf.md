# Story 10.8 — Fix Logo Quebrando no PDF

**Status:** Ready for Review
**Epic:** 10 — Revisao de Propostas Comerciais
**Priority:** Critical
**Complexity:** Low (1-3 points)

---

## Story

Como **usuario que gera propostas**, eu quero que o logo configurado no admin apareca corretamente no PDF gerado, pois atualmente o logo subido via `/admin/propostas` (tab Branding) esta quebrando/nao aparecendo no PDF.

---

## Problem

O logo subido pelo admin no Firebase Storage (path `organizations/{orgId}/branding/logo.{ext}`) nao esta sendo renderizado corretamente no PDF gerado pelo `ProposalPdf.tsx`. Possiveis causas:
- CORS bloqueando acesso ao Firebase Storage URL
- Fallback do proxy de imagem (`/api/image-proxy`) nao funcionando
- Formato ou tamanho da imagem incompativel com html2canvas/jsPDF
- URL do logo nao sendo passada corretamente via branding prop

### Debug Log
- Causa raiz: `b.logoUrl` e `b.watermarkUrl` (URLs Firebase Storage) usadas diretamente como `<img src>` sem conversão base64. CORS bloqueia html2canvas. Rota fallback `/api/image-proxy` referenciada no código não existia.
- Fix: (1) Pré-conversão de logoUrl/watermarkUrl para base64 via `urlToBase64()` em useEffect no ProposalPdf. (2) Criação da rota `/api/image-proxy` como fallback server-side.

### Completion Notes
- TypeScript: 0 erros
- ESLint: 0 warnings
- Todas as imagens de branding agora são pré-convertidas para base64 antes de renderizar, eliminando problemas de CORS tanto no preview quanto na geração PDF
- A rota `/api/image-proxy` foi criada como fallback server-side para qualquer caso de CORS residual
- O fallback gracioso já existia: `{b.logoUrl && (...)}` — se a URL não existe, nenhuma imagem é renderizada
