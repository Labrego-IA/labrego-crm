# Story 24.3 — Redesign dos Cards de Contato no Kanban

**Status:** InReview
**Epic:** 24 — Refinamentos de UX no Funil e Ligações
**Points:** 3
**Priority:** Medium
**Dependencies:** Nenhuma

---

## Story

Como usuário do CRM, quero que os cards de contato no kanban tenham um design mais harmônico e organizado, com melhor distribuição dos elementos, para facilitar a leitura rápida das informações e ter uma experiência visual mais profissional.

---

## Problem

O card atual (`KanbanCard` em `components.tsx` linhas 212-411) acumulou vários elementos ao longo de diversas stories (11.x, 21.x) sem uma revisão geral de design. Os problemas atuais:

1. **Badges empilhados sem estrutura**: proposta, dealValue, leadType, leadSource, primeiro contato, dias na etapa, tempo sem contato — todos num único flex-wrap sem agrupamento lógico
2. **Retorno agendado em posição absoluta** (top-right) pode sobrepor nome longo
3. **FRT badge** (aguardando 1º contato) aparece como bloco separado sem integração visual
4. **Responsável** (assignedToName) tem espaçamento e tamanho inconsistente com o resto
5. **Informações financeiras** (proposta/dealValue) e **informações de tracking** (leadType, leadSource, datas) estão misturadas visualmente

---

## Acceptance Criteria

### AC1: Hierarquia visual clara no card
- **Given** um card de contato no kanban
- **When** o usuário visualiza o card
- **Then** as informações estão organizadas em seções visuais distintas:
  1. **Topo**: Avatar + Nome + Empresa (informação primária)
  2. **Meio**: Dados financeiros (proposta/dealValue) e tracking (datas, tempo)
  3. **Rodapé**: Badges de classificação (leadType, leadSource, ICP) e responsável
- **And** cada seção tem espaçamento consistente

### AC2: Agrupamento lógico de badges
- **Given** os badges informativos no card
- **When** renderizados
- **Then** badges financeiros (proposta, dealValue) ficam juntos
- **And** badges de tracking (dias na etapa, tempo sem contato, primeiro contato) ficam juntos
- **And** badges de classificação (leadType, leadSource) ficam juntos
- **And** cada grupo tem separação visual sutil (gap ou divider)

### AC3: Retorno agendado reposicionado
- **Given** o badge de retorno agendado (scheduledReturn)
- **When** renderizado
- **Then** está posicionado de forma que NÃO sobreponha o nome do contato
- **And** mantém visibilidade e destaque (é informação prioritária)

### AC4: Responsável integrado ao layout
- **Given** um contato com `assignedToName` definido
- **When** o card é renderizado
- **Then** o avatar/nome do responsável está integrado ao layout geral
- **And** tamanho e espaçamento são consistentes com os demais elementos

### AC5: Consistência visual entre estados
- **Given** cards em diferentes estados (normal, overdue, dragging)
- **When** renderizados
- **Then** o layout se mantém coeso em todos os estados
- **And** o card overdue mantém destaque vermelho na borda sem quebrar o layout
- **And** o card dragging mantém a estrutura visual

### AC6: Card compacto
- **Given** o redesign aplicado
- **When** comparado com o design anterior
- **Then** o card NÃO é significativamente mais alto/largo
- **And** informações menos prioritárias podem ser truncadas
- **And** o kanban mantém boa densidade de cards por coluna

## Scope

### IN Scope
- Reorganização do layout interno do `KanbanCard`
- Agrupamento lógico de badges e informações
- Ajuste de espaçamento, padding e tipografia
- Reposicionamento do badge de retorno agendado
- Integração visual do responsável

### OUT of Scope
- Alteração de funcionalidade (drag & drop, click, etc.)
- Novos campos ou informações no card
- Alteração no `UnassignedCard` (mantém como está)
- Alteração no `TableRow` ou outras views (tabela, calendário)
- Redesign das colunas do kanban

## Dev Notes

### Estrutura proposta do card
```
┌──────────────────────────────────┐
│ [📅 Retorno] ......... [IN] [🔗] │  ← topo: retorno + classificação
│                                  │
│ [👤 Avatar]  Nome do Contato     │  ← identidade
│              Empresa LTDA        │
│              👥 2 sócios         │
│                                  │
│ 💰 R$ 50K  📋 R$ 45K Aprovada   │  ← financeiro
│                                  │
│ ⏱ 5d etapa · 💬 2d · 📞 15/01  │  ← tracking/datas
│                                  │
│ ⏰ Aguardando 1º contato (3h)   │  ← FRT (se aplicável)
│ 👤 João Silva                    │  ← responsável
└──────────────────────────────────┘
```

### Princípios de design
1. **Espaçamento**: `gap-2` entre seções, `gap-1` dentro de seções
2. **Tipografia**: Nome em `font-semibold text-sm`, resto em `text-xs`
3. **Cores**: Manter paleta existente, apenas reorganizar
4. **Truncate**: Usar `truncate` em textos longos (nome, empresa)
5. **Compacidade**: Usar `text-[10px]` para badges secundários

### Componente atual a ser refatorado
`KanbanCard` em `src/app/funil/components.tsx` (linhas 212-411)

### Manter funcionalidade
- `Draggable` wrapper — não alterar
- `onClick` handler — não alterar
- Props interface — não alterar (mesmos dados, layout diferente)
- Estados (dragging, overdue) — manter classes condicionais
