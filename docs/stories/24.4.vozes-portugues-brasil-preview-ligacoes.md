# Story 24.4 — Vozes em Português do Brasil no Preview de Ligações

**Status:** InReview
**Epic:** 24 — Refinamentos de UX no Funil e Ligações
**Points:** 2
**Priority:** High
**Dependencies:** Nenhuma

---

## Story

Como usuário do CRM, quero que o preview de vozes do agente de ligações sempre toque em português do Brasil, para avaliar corretamente como a voz soará nas chamadas com meus clientes, em vez de ouvir amostras em inglês.

---

## Problem

O preview de vozes no `VoiceSelector` tem dois caminhos:
1. **Com API key ElevenLabs**: Gera TTS em PT-BR via `/api/vapi/voices/preview` — funciona corretamente em português
2. **Sem API key ElevenLabs** (fallback): Usa as `previewUrl` estáticas da lista curada `FALLBACK_PT_VOICES` — essas URLs apontam para samples padrão da ElevenLabs que são **em inglês**

O caminho 2 é o mais comum (quando o usuário ainda não configurou a chave ElevenLabs), resultando em previews em inglês para todos os usuários novos. As vozes curadas (Lily, Liam, Sarah, Will, Daniel) são multilíngues e falam PT-BR, mas seus samples de preview hospedados na ElevenLabs são em inglês.

**Código do problema** (`VoiceSelector.tsx` linhas 110-118):
```typescript
// Fallback: use static previewUrl (English)
if (voice.previewUrl) {
  const audio = new Audio(voice.previewUrl)
  audio.play()
}
```

## Acceptance Criteria

### AC1: Preview SEMPRE em português do Brasil
- **Given** o seletor de vozes exibido na configuração de ligações
- **When** o usuário clica no botão de play/preview de qualquer voz
- **Then** o áudio reproduzido é em português do Brasil
- **And** isso funciona independente de ter ou não API key ElevenLabs configurada

### AC2: Vozes curadas com sotaque brasileiro
- **Given** a lista de vozes fallback (`FALLBACK_PT_VOICES`)
- **When** exibida ao usuário
- **Then** todas as vozes listadas têm preview em PT-BR
- **And** os nomes e descrições estão em português
- **And** as vozes reproduzem áudio com sotaque brasileiro natural

### AC3: Fallback robusto de preview
- **Given** o endpoint TTS `/api/vapi/voices/preview` falhar (sem API key)
- **When** o VoiceSelector tenta reproduzir preview
- **Then** usa os áudios PT-BR fallback (locais ou pré-gerados)
- **And** NÃO reproduz áudio em inglês em nenhuma circunstância

### AC4: Experiência fluida
- **Given** o usuário navegando pelas vozes
- **When** clica play em uma voz
- **Then** o áudio carrega em tempo razoável (< 3 segundos)
- **And** mostra indicador de loading enquanto carrega
- **And** pode pausar/trocar de voz durante reprodução

## Scope

### IN Scope
- Substituir/atualizar preview URLs em inglês por áudios em PT-BR
- Gerar ou obter áudios de preview em PT-BR para as vozes curadas
- Atualizar `FALLBACK_PT_VOICES` em `/api/vapi/voices/route.ts`
- Garantir que o fallback no `VoiceSelector.tsx` reproduz PT-BR

### OUT of Scope
- Alteração no fluxo TTS com API key (já funciona em PT-BR)
- Adicionar novas vozes além das curadas existentes
- Alteração no `AgentWizard` ou fluxo de configuração
- Alteração na execução de chamadas reais (makeVapiCall)

## Dev Notes

### Abordagem recomendada: Áudios PT-BR hospedados localmente

**Opção A — Gerar e hospedar localmente (recomendada)**:
1. Usar a ElevenLabs API (dev key) para gerar áudios PT-BR para cada voz curada
2. Salvar como `/public/audio/voices/{voiceId}-pt-br.mp3`
3. Atualizar `previewUrl` na `FALLBACK_PT_VOICES` para `/audio/voices/{voiceId}-pt-br.mp3`

**Opção B — Buscar vozes nativas PT-BR na ElevenLabs**:
1. Pesquisar na ElevenLabs Voice Library vozes criadas nativamente em PT-BR
2. Essas vozes teriam previews nativos em português
3. Substituir a lista curada por essas vozes

**Opção C — Usar Web Speech API como fallback de emergência**:
1. Se nenhum áudio disponível, usar `speechSynthesis.speak()` com `lang: 'pt-BR'`
2. Qualidade inferior, mas garante PT-BR
