# Story 24.1 — Dropdown Inline de Funil no Painel de Detalhe

**Status:** InReview
**Epic:** 24 — Refinamentos de UX no Funil e Ligações
**Points:** 3
**Priority:** High
**Dependencies:** Story 21.2 (implementação original de mover entre funis)

---

## Story

Como usuário do CRM, quero mudar o funil de um contato usando um dropdown inline no header do painel de detalhe (no mesmo padrão visual do seletor de etapa), para ter uma experiência mais rápida e consistente em vez do sistema atual de selects em bloco separado.

---

## Problem

Atualmente, "Mover para outro funil" é um bloco violet separado na parte inferior do painel lateral com 3 elementos (select funil → select etapa → botão confirmar). Essa abordagem é:
1. Visualmente desconectada do contexto do contato
2. Requer 3 cliques/interações para completar
3. Inconsistente com o seletor de etapa (que é um badge clicável inline no header)

O seletor de etapa (badge pill com `ChevronDownIcon`, dropdown flutuante com lista) é o padrão UX ideal já estabelecido no sistema. O seletor de funil deve seguir o mesmo padrão.

---

## Acceptance Criteria

### AC1: Dropdown inline de funil no header do painel
- **Given** o painel de detalhe de um cliente aberto
- **When** o usuário olha para o header (abaixo do nome do contato)
- **Then** vê um badge/pill clicável mostrando o nome do funil atual
- **And** o badge tem `ChevronDownIcon` indicando que é clicável
- **And** está posicionado próximo ao badge de etapa (acima ou ao lado)
- **And** usa estilo visual coerente com o badge de etapa (cores, tamanho, espaçamento)

### AC2: Dropdown flutuante com lista de funis
- **Given** o badge de funil clicável
- **When** o usuário clica no badge
- **Then** abre um dropdown flutuante com a lista de funis da organização
- **And** cada opção mostra o nome do funil com indicador de cor
- **And** o funil atual aparece destacado com check icon
- **And** clicar fora do dropdown o fecha

### AC3: Fluxo de mudança de funil com seleção de etapa
- **Given** o dropdown de funis aberto
- **When** o usuário seleciona um funil diferente do atual
- **Then** o dropdown muda para mostrar as etapas do funil selecionado
- **And** cada etapa mostra nome + cor + probabilidade %
- **And** ao selecionar uma etapa, confirma a mudança automaticamente (sem botão extra)

### AC4: Lógica de movimentação preservada
- **Given** o usuário confirmou a mudança de funil + etapa
- **When** a movimentação é processada
- **Then** atualiza no Firestore: `funnelId`, `funnelStage`, `funnelStageUpdatedAt`
- **And** limpa campos de cadência (`currentCadenceStepId`, `lastCadenceActionAt`, `lastCadenceStepResponded`)
- **And** reseta `closingProbability` via `deleteField()`
- **And** adiciona log na subcollection `clients/{id}/logs`
- **And** exibe toast de sucesso
- **And** o cliente desaparece da view atual do kanban

### AC5: Remover bloco violet antigo
- **Given** o novo dropdown inline implementado
- **When** o painel de detalhe é exibido
- **Then** o bloco violet "Mover para outro funil" (linhas 5734-5778) NÃO existe mais
- **And** toda a funcionalidade está no novo dropdown inline

---

## Scope

### IN Scope
- Badge/pill de funil no header do painel de detalhe
- Dropdown flutuante com lista de funis (mesmo padrão da etapa)
- Sub-dropdown de etapas ao selecionar funil diferente
- Remoção do bloco violet antigo
- Manter toda lógica de movimentação existente (handler, Firestore, log, toast)

### OUT of Scope
- Alteração no drag & drop do kanban
- Alteração na lógica de reset de probabilidade (já existe)
- Histórico visual de movimentações entre funis
- Mover em massa entre funis

## Dev Notes

### Padrão visual de referência (dropdown de etapa existente)
O dropdown de etapa no header (`page.tsx` linhas 5419-5488) usa:
- `button` com classes `inline-flex items-center gap-1.5 px-2.5 py-1 rounded-full text-xs font-medium`
- `ChevronDownIcon` com rotação no open state
- Dropdown absoluto com `bg-white rounded-xl shadow-xl border border-slate-200 py-2 max-h-64 overflow-y-auto`
- Items com `w-full flex items-center gap-3 px-3 py-2`
- Color indicator (circle) + nome + check se selecionado

### Layout sugerido no header
```
[Avatar] Nome do Contato
         Empresa
         [🔵 Funil Comercial ▾] [🟢 Qualificação ▾] ← badges inline
```

### Estados do dropdown
1. **Fechado**: badge/pill mostrando nome do funil atual
2. **Aberto - Lista de funis**: lista de funis disponíveis
3. **Aberto - Lista de etapas**: ao selecionar funil diferente, mostra etapas do funil destino

### Reutilizar

- States: `moveFunnelTarget`, `moveFunnelStage`, `moveFunnelStages`, `movingFunnel`
