# Story 21.3 — Página de Projeção de Vendas

**Status:** Ready
**Epic:** 21 — Pipeline de Vendas & KPIs de Funil
**Points:** 8
**Priority:** High
**Dependencies:** Story 21.1 (formatação), Story 21.2 (closingProbability, getClientProbability)

---

## Story

Como usuário do CRM, quero uma página dedicada de Projeção de Vendas acessível pelo sidebar, onde posso visualizar todos os contatos com valor de negócio e probabilidade acima de zero em uma tabela interativa por funil, com edição inline de campos-chave e ordenação por coluna, para ter uma visão consolidada da expectativa de receita.

---

## Acceptance Criteria

### AC1: Nova página no sidebar
- **Given** o usuário autenticado
- **When** acessa o sidebar
- **Then** vê um novo item "Projeção de Vendas" com ícone `ChartBarIcon` (ou `PresentationChartLineIcon`)
- **And** a rota é `/projecao-vendas`
- **And** está posicionado após "Análises & Insights" no menu

### AC2: Tabela agrupada por funil
- **Given** a página carregada
- **When** existem contatos com `dealValue > 0` OU probabilidade efetiva > 0
- **Then** exibe uma seção por funil com:
  - Cabeçalho com nome do funil + cor + count de contatos
  - Tabela com os contatos daquele funil que atendem ao filtro
- **And** funis sem contatos elegíveis não são exibidos
- **And** usa `onSnapshot` com error callback para dados real-time

### AC3: Colunas da tabela
A tabela deve exibir as seguintes colunas:
1. **Nome do Cliente** — texto, link para `/contatos/{id}`
2. **Etapa Atual** — nome da etapa, editável via select inline
3. **Probabilidade (%)** — número, editável inline (onBlur, 0-100)
4. **Valor do Negócio (R$)** — número, editável inline (onBlur)
5. **Valor Esperado (R$)** — calculado: `dealValue × probability / 100` (read-only)
6. **Dias na Etapa** — calculado a partir de `funnelStageUpdatedAt`
7. **Último Contato** — data formatada de `lastFollowUpAt` ou "Sem contato"

### AC4: Edição inline
- **Given** o usuário clica em um campo editável (etapa, probabilidade, valor)
- **When** altera o valor
- **Then** salva no Firestore via `updateDoc` com padrão onBlur
- **And** exibe toast de confirmação
- **And** o valor esperado recalcula automaticamente
- **And** a edição de etapa carrega as etapas daquele funil no select

### AC5: Ordenação por coluna
- **Given** a tabela renderizada
- **When** o usuário clica no header de qualquer coluna
- **Then** ordena a tabela por aquela coluna (toggle asc/desc)
- **And** exibe seta indicando direção da ordenação
- **And** ordenação default: valor esperado desc

### AC6: Totalizadores por funil
- **Given** uma seção de funil com contatos
- **When** renderizada
- **Then** exibe na última linha (footer) da tabela:
  - **Soma Valor do Negócio**: total de dealValue
  - **Soma Valor Esperado**: total de valor esperado
- **And** valores formatados em BRL

### AC7: Totalizador geral
- **Given** a página completa
- **When** renderizada
- **Then** exibe no topo um resumo geral:
  - Total de contatos no pipeline
  - Valor total de negócios (todos os funis)
  - Valor esperado total (todos os funis)
- **And** formatado com cards estilo KPI

## Dev Notes

### Estrutura de dados
A página precisa carregar:
1. **Funis**: `organizations/{orgId}/funnels`
2. **Stages**: `funnelStages` where `orgId == orgId`
3. **Clientes**: `clients` where `orgId == orgId` — filtrar client-side por `dealValue > 0` OR `closingProbability > 0` OR stage com probability > 0

### Cálculos
```typescript
// Probabilidade efetiva (reusar helper da 21.2)
const probability = getClientProbability(client, stage)

// Valor esperado
const expectedValue = (client.dealValue || 0) * probability / 100

// Dias na etapa
const daysInStage = client.funnelStageUpdatedAt
  ? Math.floor((Date.now() - new Date(client.funnelStageUpdatedAt).getTime()) / 86400000)
  : null

// Último contato
const lastContact = client.lastFollowUpAt
  ? new Date(client.lastFollowUpAt).toLocaleDateString('pt-BR')
  : 'Sem contato'
```

### Edição inline pattern
Usar o padrão já estabelecido: `defaultValue` + `onBlur` + `key={client.id}` para inputs numéricos. Para select de etapa, usar `onChange` direto (não precisa onBlur).

### Ordenação
```typescript
type SortField = 'name' | 'stage' | 'probability' | 'dealValue' | 'expectedValue' | 'daysInStage' | 'lastContact'
type SortDir = 'asc' | 'desc'
const [sortField, setSortField] = useState<SortField>('expectedValue')
const [sortDir, setSortDir] = useState<SortDir>('desc')
```
