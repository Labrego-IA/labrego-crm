# Story 12.5 — Substituir Tab Knowledge + Migracao de Configs Existentes

**Status:** Ready for Review
**Epic:** 12 — Wizard Gamificado de Configuracao do Agente de Voz
**Priority:** High
**Complexity:** Medium (5 points)
**Depends On:** Story 12.2, Story 12.3, Story 12.4

---

## Story

Como **usuario que ja tem um agente configurado no sistema antigo**, eu quero que a nova experiencia do wizard substitua a aba antiga sem perder minha configuracao existente, para que a transicao seja transparente e eu nao precise reconfigurar tudo do zero.

---

## Problem

Usuarios existentes ja tem `agentKnowledge` preenchido (campos individuais e/ou `systemPrompt` escrito manualmente). Se substituirmos a aba "Conhecimento" pelo wizard sem migracao, eles perdem a configuracao e o agente para de funcionar corretamente.

**Mapeamento de migracao:**
```typescript
function migrateKnowledgeToWizard(k: CallAgentKnowledge): AgentWizardAnswers {
  return {
    agentName: k.agentName || '',
    agentRole: k.agentRole || '',
    companyName: k.companyName || '',
    toneDescription: k.toneOfVoice || '',
    whatYouSell: k.productsServices || '',
    idealCustomer: k.targetAudience || '',
    differentials: k.competitiveDifferentials || '',
    valueProposition: k.valueProposition || '',
    openingApproach: k.firstMessage || '',
    hookStrategy: '',  // Nao tem equivalente
    discoveryQuestions: [],  // Nao tem equivalente
    qualificationCriteria: '',  // Nao tem equivalente
    solutionBridge: '',  // Nao tem equivalente
    specialistName: '',  // Nao tem equivalente
    meetingDuration: 30,
    objections: k.commonObjections?.map(o => ({ objection: o.objection, response: o.response })) || [],
    forbiddenWords: k.forbiddenPhrases?.join(', ') || '',
    keyExpressions: k.keyPhrases?.join(', ') || '',
    behaviorRules: '',
    completedPhases: [],  // Sera recalculado
    strengthScore: 0,  // Sera recalculado
    lastUpdated: new Date().toISOString(),
  }
}
```

**Fases com dados migrados vs vazias:** Apos migracao, as fases que tiveram dados preenchidos devem ser marcadas em `completedPhases` e o `strengthScore` recalculado. Fases sem equivalente (3 parcial, 4, 5 parcial) ficam vazias para o usuario completar.
