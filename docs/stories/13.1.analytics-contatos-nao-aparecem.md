# Story 13.1 — Contatos nao Aparecem na Pagina de Analytics

**Status:** Ready for Review
**Epic:** 13 — Bug Fixes e Melhorias de Estabilidade
**Priority:** Critical
**Complexity:** Medium (5 points)
**Depends On:** None

---

## Story

Como **usuario que acessa a pagina de Analytics**, eu quero ver os contatos e metricas corretamente, porque atualmente a pagina nao exibe nenhum contato, impedindo a analise de dados do funil.

---

## Problem

A pagina `/analytics` nao esta exibindo contatos. As possiveis causas identificadas sao:
1. **orgId nulo/undefined:** O layout carrega orgId assincronamente e a pagina pode renderizar antes do orgId estar disponivel
2. **Falha silenciosa nas queries:** Erros no Firestore sao logados no console mas nao mostrados ao usuario — a pagina fica em loading infinito
3. **Clientes sem `funnelStage`:** Contatos sem o campo `funnelStage` sao silenciosamente ignorados nos calculos
4. **Funis/stages vazios:** Se nao ha funis ou stages configurados, nenhum contato aparece
5. **useRef nao dispara re-render:** Os dados sao armazenados em `useRef` com `loading` como proxy para forcar recalculo — se o timing falhar, os memos nao recalculam

---

## Acceptance Criteria

- [ ] AC1: DADO que o orgId esta disponivel e ha contatos no Firestore, QUANDO a pagina de analytics carrega, ENTAO os contatos aparecem nas metricas e tabelas
- [ ] AC2: DADO que o orgId esta carregando, QUANDO a pagina renderiza, ENTAO mostra skeleton/loading ate o orgId estar disponivel
- [ ] AC3: DADO que a query do Firestore falha, QUANDO ocorre erro, ENTAO mostra mensagem de erro ao usuario (nao apenas no console)
- [ ] AC4: DADO que nao ha contatos no Firestore, QUANDO a pagina carrega, ENTAO mostra estado vazio ("Nenhum contato encontrado")
- [ ] AC5: DADO que contatos existem mas sem campo `funnelStage`, QUANDO analytics calcula, ENTAO esses contatos aparecem na contagem geral (mesmo sem associacao a stage)
- [ ] AC6: DADO que nao ha funis configurados, QUANDO a pagina carrega, ENTAO mostra mensagem orientativa ("Configure um funil para ver analytics completo")
- [ ] AC7: TypeScript compila sem erros
- [ ] AC8: ESLint passa sem warnings

## Dev Notes

**Arquivo principal:** `src/app/analytics/page.tsx` (~1.004 linhas)
**Calculos:** `src/lib/analyticsCalculations.ts` (~377 linhas)
**Contexto org:** `src/contexts/CrmUserContext.tsx`

**Fluxo de dados atual:**
```
useCrmUser() → orgId
  → getDocs(organizations/{orgId}/clients)
  → getDocs(organizations/{orgId}/funnels)
  → getDocs(organizations/{orgId}/funnels/{id}/stages)
  → clientsRef.current = clients
  → loading = false
  → useMemo recalcula (depende de [selectedFunnel, loading])
```

**Pontos de investigacao prioritarios:**

1. **Timing do orgId (mais provavel):**
```typescript
// Linha 168
const { orgId } = useCrmUser()
// Linha 184
if (!orgId) return  // Se orgId nao esta pronto, dados nunca carregam
```
Solucao: adicionar `orgId` como dependencia do useEffect que chama `loadData`.

2. **useRef vs useState:**
```typescript
// Linha 177-180 — dados em useRef (nao causam re-render)
const clientsRef = useRef<Client[]>([])
// Linha 231 — useMemo usa loading como proxy
}, [selectedFunnel, loading])  // eslint-disable-line
```
Solucao: considerar migrar para useState ou adicionar um counter de re-render.

3. **Erro silencioso:**
```typescript
// Linha 210-211
catch (err) {
  console.error('Failed to load analytics data:', err)
  // Nenhum feedback visual!
}
```
Solucao: setState com mensagem de erro.

4. **Contatos sem funnelStage ignorados:**
```typescript
// analyticsCalculations.ts linha 61-64
const stageId = c.funnelStage as string
const stage = stages.find((s) => s.id === stageId)
if (!stage) continue  // SILENCIOSAMENTE IGNORADO
```
Solucao: incluir esses contatos em uma categoria "Sem stage" ou na contagem geral.
