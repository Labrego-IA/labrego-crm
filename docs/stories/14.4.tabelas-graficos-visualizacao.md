## Story

Como **usuario que analisa dados no CRM**, eu quero que tabelas, graficos e indicadores visuais sigam a nova identidade laranja/branco/preto, para que a visualizacao de dados seja coerente com a nova marca.

---

## Problem

As tabelas usam `--gradient-header` (que sera atualizado em 14.1) mas as linhas alternadas usam `bg-primary/5` e `hover:bg-primary/10` que precisam ser verificadas com laranja. Os graficos (Recharts) usam cores definidas inline em cada pagina. Os funnel stage colors sao semanticos e nao devem mudar, mas o palette geral de charts precisa harmonizar com laranja.

## Dev Notes

**Linhas alternadas com laranja:**
O `even:bg-primary/5` com laranja pode ficar muito quente. Se necessario, trocar para:
```css
/* Em vez de laranja sutil, usar neutro */
:where(.app-table tbody tr) {
  @apply transition-colors odd:bg-white even:bg-neutral-50 hover:bg-primary/10;
}
```
Testar visualmente e decidir.

**Cores de graficos (Recharts):**
Buscar nas paginas analytics e conversao por `fill=`, `stroke=`, `colors=` com valores hardcoded indigo/violet e substituir:
```tsx
// ANTES:
<Area fill="#4f46e5" stroke="#4f46e5" />

// DEPOIS:
<Area fill="#F97316" stroke="#EA580C" />
```

**Palette de charts sugerida:**
```
Primario:   #F97316 (laranja 500)
Secundario: #FB923C (laranja 400)
Terciario:  #FDBA74 (laranja 300)
Accent:     #171717 (preto)
Neutral:    #A3A3A3 (cinza)
```

**Funnel stage colors (NAO ALTERAR):**
Estas cores sao mapeadas semanticamente por stage name e devem permanecer:
- Novo Lead → Blue
- Qualificacao → Amber
- Proposta → Purple
- Negociacao → Orange (coincide com primary — OK)
- Fechado Ganho → Emerald
- Fechado Perdido → Red
