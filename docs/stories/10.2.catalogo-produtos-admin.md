# Story 10.2 — Catalogo de Produtos no Admin

**Status:** Done
**Epic:** 10 — Revisao de Propostas Comerciais
**Priority:** High
**Complexity:** Medium (5-8 points)

---

## Story

Como **admin de uma organizacao**, eu quero cadastrar e gerenciar produtos (nome, preco, valor/hora, margem, impostos, cronograma) e logos de clientes em uma area administrativa, para que esses dados alimentem automaticamente o formulario de criacao de propostas.

---

## Problem

Produtos e logos eram inseridos manualmente a cada proposta. Nao existia um catalogo centralizado por organizacao, causando inconsistencia de precos e retrabalho.

---

## Acceptance Criteria

- [x] AC1: A pagina `/admin/propostas` possui sub-tabs: Branding, Produtos, Logos de Clientes, Configuracoes, Estrutura do PDF
- [x] AC2: Tab Produtos permite CRUD completo (criar, editar, deletar) de produtos
- [x] AC3: Cada produto tem: nome, descricao, preco, valor/hora, margem, imposto, cronograma (etapas + dias)
- [x] AC4: Produtos sao salvos na colecao `products` com filtro por `orgId`
- [x] AC5: Tab Logos permite upload multiplo de logos de clientes
- [x] AC6: Logos sao salvos no Firebase Storage e registrados na colecao `logos`
- [x] AC7: Tab Branding foi extraida para componente separado (PropostasBrandingTab)
- [x] AC8: Componente Tabs reutilizavel usado para navegacao entre sub-tabs
- [x] AC9: TypeScript compila sem erros
- [x] AC10: ESLint passa sem warnings
