# Story 14.1 — Design Tokens: Paleta Laranja/Branco/Preto

**Status:** Ready for Review
**Epic:** 14 — Redesign Visual: Identidade Labrego (Laranja/Branco/Preto)
**Priority:** Critical
**Complexity:** Medium (5 points)
**Depends On:** None

---

## Story

Como **usuario do CRM**, eu quero que o sistema tenha uma identidade visual marcante com laranja, branco e preto, para que a experiencia seja profissional, moderna e memoravel.

---

## Problem

Hoje o CRM usa indigo/violet (#4f46e5) como cor primaria — uma escolha generica que nao diferencia a marca. As cores estao espalhadas em 3 locais (tailwind.config.js, globals.css, ui.ts) sem uma paleta coesa. A identidade visual nao transmite energia nem profissionalismo.

---

## Acceptance Criteria

- [ ] AC1: Cor primaria alterada de indigo para laranja em `tailwind.config.js` — escala completa 50-900
- [ ] AC2: CSS variables em `globals.css` `:root` atualizadas para a nova paleta laranja
- [ ] AC3: CSS variables `.dark` atualizadas para paleta laranja em dark mode
- [ ] AC4: `--gradient-header` atualizado para gradiente laranja (ex: `#EA580C → #F97316 → #FB923C`)
- [ ] AC5: `ui.ts` continua funcionando sem mudancas (usa `primary` que sera resolvido automaticamente)
- [ ] AC6: Cor `dark` atualizada de `#1E1E2F` para `#171717` (near-black puro)
- [ ] AC7: Cor `accent` atualizada para complementar laranja (ex: `#FB923C` ou `#FDBA74`)
- [ ] AC8: Cores semanticas (success, warning, error) mantidas sem alteracao
- [ ] AC9: DADO qualquer pagina do CRM, QUANDO renderizada, ENTAO botoes primarios sao laranja, nao indigo
- [ ] AC10: DADO a sidebar, QUANDO visualizada, ENTAO active state usa laranja (antes: violet)
- [ ] AC11: DADO tabelas com header gradient, QUANDO renderizadas, ENTAO gradient e laranja
- [ ] AC12: `globals.css` componentes `.btn`, `.btn-primary` usam a nova cor sem mudancas manuais (via `primary`)
- [ ] AC13: TypeScript compila sem erros
- [ ] AC14: ESLint passa sem warnings

```css
  --color-primary-400: #FB923C;
  --color-secondary: #ffffff;
  --color-black: #171717;

```
