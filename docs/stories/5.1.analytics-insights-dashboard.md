## Story

**Como** gestor de vendas ou administrador do CRM,
**Quero** acessar dashboards analíticos com visões de aging de contatos, perfil de clientes fechados, oportunidades no funil, e conversão segmentada por múltiplos parâmetros,
**Para que** eu consiga identificar padrões, gargalos e oportunidades no pipeline e tomar decisões estratégicas baseadas em dados.

## Acceptance Criteria

### Bloco A — Infraestrutura e Navegação

#### AC-1: Ativação da rota `/analytics` no sidebar
**Given** o módulo de analytics está implementado
**When** o usuário com plano Pro acessa o CRM
**Then** o item "Análises & Insights" no sidebar está ativo (sem badge "Em breve")
**And** ao clicar, navega para `/analytics`
**And** se o plano NÃO é Pro, exibe `PlanGate` com `UpgradePrompt`

#### AC-2: Layout com abas de visões analíticas
**Given** o usuário acessa `/analytics`
**When** a página carrega
**Then** vê um layout com abas:
1. **Visão Geral** (default)
2. **Aging de Contatos**
3. **Perfil de Clientes**
4. **Oportunidades**
5. **Conversão por Parâmetro**
**And** filtros globais no topo: período (7d/30d/90d/12m/custom), funil (todos ou específico)

### Bloco B — Visão Geral (Dashboard KPI)

#### AC-3: KPIs principais
**Given** o usuário está na aba "Visão Geral"
**When** os dados carregam
**Then** exibe cards com:
- **Total de contatos** na base (com variação % vs período anterior)
- **Leads novos** no período selecionado
- **Contatos ativos** (status = Ativo) vs inativos
- **Taxa de conversão geral** (leads → clientes fechados no período)
- **Tempo médio de conversão** (days from firstContactAt to status=Ativo)
- **Contatos sem atividade** (>30 dias sem follow-up)

#### AC-4: Gráfico de evolução temporal
**Given** o usuário está na aba "Visão Geral"
**When** os dados carregam
**Then** exibe um gráfico de linha (Recharts `LineChart`) com:
- Eixo X: tempo (dias/semanas/meses conforme período selecionado)
- Linhas: novos leads, contatos fechados, leads perdidos
**And** hover exibe tooltip com valores exatos
**And** legenda interativa (clicar na legenda esconde/mostra linha)

### Bloco C — Aging de Contatos

#### AC-5: Mapa de calor Aging × Etapa
**Given** o usuário está na aba "Aging de Contatos"
**When** os dados carregam
**Then** exibe uma tabela estilo heatmap com:
- **Linhas**: etapas do funil (ordenadas por ordem no funil)
- **Colunas**: faixas de tempo desde o último contato (0-7d, 8-15d, 16-30d, 31-60d, 61-90d, 90d+)
- **Células**: quantidade de contatos naquela etapa × faixa de tempo
- **Cores**: gradiente de verde (recente) → amarelo → vermelho (>90 dias)
**And** clicar em uma célula exibe a lista de contatos daquela combinação

#### AC-6: Aging por data de criação
**Given** o usuário está na aba "Aging de Contatos"
**When** ele alterna para "Por data de criação"
**Then** exibe gráfico de barras empilhadas (Recharts `BarChart`):
- Eixo X: meses de criação
- Barras empilhadas por status atual (Lead, Lead-qualificado, Ativo, Inativo)
- Mostra quantos leads de cada mês chegaram a cada status
**And** permite identificar "safras" de leads com melhor conversão

#### AC-7: Indicador de leads "esfriando"
**Given** os dados de aging estão carregados
**When** o usuário visualiza
**Then** vê destaque com:
- Total de contatos sem atividade há 30+ dias
- Total de contatos sem atividade há 60+ dias
- Alerta visual com botão "Ver contatos" que redireciona ao funil filtrado

### Bloco D — Perfil de Clientes Fechados

#### AC-8: Análise de perfil dos clientes ativos (fechados)
**Given** o usuário está na aba "Perfil de Clientes"
**When** os dados carregam
**Then** exibe análise dos contatos com `status = 'Ativo'`:

**Gráficos de distribuição:**
- **Por segmento/indústria** (`industry`): gráfico de pizza/donut (Recharts `PieChart`)
- **Por porte de empresa** (`porte_empresa`): barras horizontais
- **Por origem do lead** (`leadSource`): barras horizontais
- **Por tipo do lead** (`leadType`): Inbound vs Outbound (donut)
- **Por estado** (`estado`): mapa ou barras horizontais top 10
- **Por tipo de empresa** (`tipo`): barras horizontais
- **Por natureza jurídica** (`natureza_juridica`): barras horizontais

**Cada gráfico mostra:** quantidade absoluta + percentual

#### AC-9: Comparativo perfil ativo vs pipeline
**Given** o usuário está na aba "Perfil de Clientes"
**When** ele ativa o toggle "Comparar com pipeline"
**Then** cada gráfico mostra lado a lado:
- Distribuição dos clientes **ativos** (fechados)
- Distribuição dos leads **no pipeline** (não fechados)
**And** permite identificar: "Nossos melhores clientes vêm de X segmento, mas estamos prospectando pouco nesse segmento"

### Bloco E — Oportunidades no Funil

#### AC-10: Valor do pipeline por etapa
**Given** o usuário está na aba "Oportunidades"
**When** os dados carregam
**Then** exibe um funil visual com:
- Cada etapa mostrando: quantidade de contatos, % do total
- Barras proporcionais ao volume de contatos
- Taxa de conversão entre etapas adjacentes (seta com %)
- Tempo médio de permanência em cada etapa

#### AC-11: Ranking de oportunidades "quentes"
**Given** o usuário está na aba "Oportunidades"
**When** os dados carregam
**Then** exibe tabela de "Top Oportunidades" com:
- Contatos nas etapas de maior probabilidade (probability > 50%)
- Colunas: nome, empresa, etapa atual, probabilidade, dias na etapa, responsável, última atividade
- Ordenado por probabilidade × tempo na etapa (urgência)
- Destaque visual para contatos atrasados (dias > maxDays da etapa)
**And** clicar no contato redireciona para `/contatos/[id]`

#### AC-12: Etapas com gargalo
**Given** os dados de oportunidades estão carregados
**When** o usuário visualiza
**Then** identifica automaticamente etapas com gargalo:
- Etapas onde tempo médio > maxDays configurado
- Etapas com alta taxa de saída negativa
- Destaque visual (card vermelho) com sugestão: "X contatos estão há mais de Y dias em [Etapa]"

### Bloco F — Conversão por Parâmetro

#### AC-13: Seletor de dimensão de análise
**Given** o usuário está na aba "Conversão por Parâmetro"
**When** a página carrega
**Then** vê um seletor de dimensão com as opções:
- Por origem do lead (`leadSource`)

- Por segmento/indústria (`industry`)
- Por porte de empresa (`porte_empresa`)

  - Barra total (leads entrados)

  - Dimensão (ex: "Google")

## Estratégia de Dados e Performance

### Agregação Client-Side vs Server-Side
Firestore **não suporta** agregação nativa (GROUP BY, COUNT, AVG). Duas abordagens:

**Abordagem recomendada: Snapshot + agregação client-side**
```typescript
// Buscar todos os contatos da org (uma vez por sessão)
const clientsRef = collection(db, `organizations/${orgId}/clients`)
const snapshot = await getDocs(clientsRef)
const clients = snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() }))

// Agregar em memória
const byLeadSource = groupBy(clients, 'leadSource')
const conversionBySource = Object.entries(byLeadSource).map(([source, contacts]) => ({
  source,
  total: contacts.length,
  converted: contacts.filter(c => c.status === 'Ativo').length,
  rate: converted / total
}))
```

**Limites:** Para orgs com >10.000 contatos (plano Pro max), usar paginação ou cache.

**Cache strategy:**
- Carregar dados uma vez ao entrar em `/analytics`
- Manter em state/ref durante a sessão
- Filtros globais filtram os dados em memória (sem re-fetch)
- Botão "Atualizar dados" para refresh manual

### Helpers de Cálculo
Criar `src/lib/analyticsCalculations.ts` com funções puras:
```typescript
// Exemplos
calcAgingMatrix(clients, stages, dateField)
calcConversionByDimension(clients, dimension, targetStatus)
calcTimeToConversion(clients)
groupByPeriod(items, dateField, periodType)
calcBottleneckStages(clients, stages)
```

```yaml
mode: light
max_iterations: 2
severity_filter: [CRITICAL, HIGH]
behavior:
  CRITICAL: auto_fix
  HIGH: auto_fix (iteration < 2) else document_as_debt
  MEDIUM: document_as_debt
  LOW: ignore
```

- **CRITICAL**: Verificar que `orgId` está em todas as queries (multi-tenant isolation)
