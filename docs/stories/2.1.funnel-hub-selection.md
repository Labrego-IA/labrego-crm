## Story

**Como** usuário do CRM com permissão de acesso ao funil,
**Quero** ver uma tela de seleção de funis ao acessar `/funil`, onde posso criar, visualizar e selecionar meus funis,
**Para que** eu possa organizar diferentes pipelines de vendas de forma independente e acessar rapidamente o funil desejado.

---

## Contexto

### Estado Atual
- A rota `/funil` carrega diretamente o Kanban/tabela com os contatos do funil default
- O sistema já suporta múltiplos funis (`src/lib/funnels.ts` com CRUD completo)
- Tipos já definidos em `src/types/funnel.ts` (Funnel, FunnelColumn)
- Hook `useVisibleFunnels` já filtra funis por visibilidade do membro
- Limites por plano já existem: Basic=1, Standard=3, Pro=10 (`src/types/plan.ts`)
- Permissão `canManageFunnels` já existe (admin/manager = true, seller/viewer = false)
- A página `funil/page.tsx` tem ~6.650 linhas com Kanban, tabela, calendário e log de atividades

### O que Muda
- `/funil` passa a exibir primeiro um **Funnel Hub** (grid de cards dos funis)
- Ao clicar em um funil, navega para `/funil/[funnelId]` com o conteúdo atual (Kanban/tabela/etc.)
- Usuários com permissão podem criar funis até o limite do plano da organização
- Ao atingir o limite, exibe `UpgradePrompt` orientando contato com super admin

## Acceptance Criteria

### AC-1: Funnel Hub como página inicial de `/funil`
**Given** um usuário autenticado com acesso ao funil
**When** ele acessa `/funil`
**Then** vê uma tela com grid de cards representando os funis visíveis para ele
**And** cada card exibe: nome do funil, cor, quantidade de contatos, se é o default, e data de criação

### AC-2: Card de funil com informações resumidas
**Given** o Funnel Hub está carregado
**When** o usuário visualiza um card de funil
**Then** o card exibe:
- Nome do funil
- Cor identificadora (barra lateral ou borda)
- Número total de contatos no funil
- Badge "Default" se for o funil padrão
- Número de etapas configuradas
- Indicador de contatos atrasados (se houver)

### AC-3: Navegação para detalhes do funil
**Given** o usuário está no Funnel Hub
**When** ele clica em um card de funil
**Then** é redirecionado para `/funil/[funnelId]`
**And** a página exibe o Kanban/tabela/calendário/log com os contatos daquele funil específico
**And** há um botão/link para voltar ao Funnel Hub

### AC-4: Criação de novo funil
**Given** o usuário tem permissão `canManageFunnels` (admin ou manager)
**And** a organização NÃO atingiu o limite de funis do plano
**When** ele clica no botão "Novo Funil" no Funnel Hub
**Then** um modal abre solicitando: nome (obrigatório), descrição (opcional), cor (seletor com 10 opções)
**And** ao confirmar, o funil é criado e aparece no grid

### AC-5: Bloqueio por limite do plano
**Given** o usuário tem permissão `canManageFunnels`
**And** a organização JÁ atingiu o limite de funis do plano (ex: Basic=1)
**When** ele tenta criar um novo funil
**Then** o botão "Novo Funil" está desabilitado ou exibe tooltip informando o limite
**And** exibe o componente `UpgradePrompt` sugerindo upgrade do plano ou contato com super admin

### AC-6: Edição e exclusão de funil pelo Hub
**Given** o usuário tem permissão `canManageFunnels`
**When** ele clica no menu de ações (três pontos) de um card de funil
**Then** pode editar nome, descrição, cor e visibilidade
**And** pode excluir o funil (com confirmação e aviso que contatos serão desvinculados)
**And** NÃO pode excluir o funil default (botão desabilitado com tooltip explicativo)

### AC-7: Visibilidade baseada em membro
**Given** um funil tem o campo `visibleTo` configurado com IDs de membros específicos
**When** um membro que NÃO está na lista acessa `/funil`
**Then** esse funil NÃO aparece no Funnel Hub dele
**And** se tentar acessar `/funil/[funnelId]` diretamente, vê mensagem "Funil não encontrado ou sem permissão"

### AC-8: Estado vazio (primeiro acesso / nenhum funil)
**Given** a organização não tem nenhum funil criado
**When** o usuário acessa `/funil`
**Then** vê um estado vazio com ilustração/ícone e mensagem "Nenhum funil criado ainda"
**And** se tem permissão, vê botão "Criar seu primeiro funil"
**And** se não tem permissão, vê mensagem "Solicite ao administrador a criação de um funil"

### AC-9: Responsividade
**Given** o usuário acessa o Funnel Hub
**When** visualiza em diferentes tamanhos de tela
**Then** o grid se adapta: 3 colunas em desktop, 2 em tablet, 1 em mobile

### AC-10: Migração da rota existente
**Given** a implementação anterior usava `/funil` para exibir diretamente o Kanban
**When** a nova versão é deployada
**Then** `/funil` mostra o Funnel Hub
**And** `/funil/[funnelId]` mostra o conteúdo que antes estava em `/funil`
**And** se a organização tem apenas 1 funil, o Hub ainda é exibido (sem auto-redirect)

## Dev Notes

### Referência de Arquitetura
- Consultar `docs/architecture/system-architecture.md` para visão geral do sistema
- O módulo Funil é identificado como **complexidade Alta** na análise arquitetural

### Estratégia de Migração
A principal complexidade está em mover o conteúdo de `funil/page.tsx` (6.650 linhas) para `funil/[funnelId]/page.tsx` sem quebrar funcionalidade:

1. **Copiar** `page.tsx` para `[funnelId]/page.tsx`
2. **Adaptar** para ler `funnelId` de `params` ao invés de usar o default
3. **Manter** `components.tsx` no mesmo nível (importável por ambas as rotas)
4. **Reescrever** `page.tsx` original como Funnel Hub (componente novo, muito menor)

### Contagem de Contatos por Funil
Para exibir o count nos cards, usar query Firestore:
```typescript
// Query contatos agrupados por funnelStage pertencente ao funil
const clientsQuery = query(
  collection(db, `organizations/${orgId}/clients`),
  where('funnelId', '==', funnelId) // ou filtrar por funnelStages do funil
);
```
**Nota:** Verificar se o campo `funnelId` existe nos clientes ou se precisa ser inferido via `funnelStage` pertencente ao funil. Caso não exista, será necessário adicionar ou usar uma abordagem de lookup.

### Testing Standards
- Testar criação de funil com limite atingido (toast de erro + UpgradePrompt)
- Testar visibilidade: membro fora do `visibleTo` não vê o funil
- Testar exclusão: contatos devem ser desvinculados (funnelStage = null)
- Testar navegação: Hub → Detalhe → Hub sem perda de estado
- Testar responsividade em 3 breakpoints
- Testar acesso direto a `/funil/[funnelId]` sem permissão
