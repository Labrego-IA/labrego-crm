# Story 9.1 — Credits Access Control: Admin Read-Only + Super Admin Exclusive Add

**Status:** Ready for Review
**Epic:** 9 — Access Control & Security
**Priority:** High
**Complexity:** Low (1-2 points)

---

## Story

Como **product owner**, eu quero que a rota `/admin/creditos` permita apenas **consultar saldo e historico** de creditos (somente leitura), removendo a capacidade de adicionar creditos, para que apenas o **super admin** (via `/super-admin/creditos`) tenha permissao de adicionar/ajustar creditos nas organizacoes.

---

## Problem

Atualmente a pagina `/admin/creditos` possui um formulario "Adicionar creditos" que permite qualquer admin da organizacao adicionar minutos de ligacao diretamente no Firestore (client-side, sem validacao server-side). Isso e um problema de seguranca e governanca:
1. **Sem controle centralizado**: Qualquer admin pode inflar creditos sem aprovacao
2. **Client-side write**: A operacao usa `updateDoc` + `addDoc` diretamente do frontend sem API route protegida
3. **Duplicidade**: A funcionalidade de adicionar creditos ja existe no super-admin (`/super-admin/creditos`) com protecao adequada via API server-side
