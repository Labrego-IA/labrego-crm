## Story

Como **usuario que configura o agente de voz**, eu quero ouvir amostras de vozes disponiveis e escolher a que melhor representa meu agente, para que o agente tenha uma voz natural e agradavel que combine com a identidade que configurei.

---

## Problem

Hoje o campo `voiceId` existe no tipo `CallRoutingVoiceAgent` mas nao e usado na UI. O usuario nao tem como saber quais vozes estao disponiveis, ouvir amostras, nem escolher uma. O resultado e que todos os agentes usam a voz padrao do VAPI, sem personalizacao.

## Dev Notes

**API Route `/api/vapi/voices`:**
```typescript
// GET - Listar vozes disponiveis
export async function GET(req: NextRequest) {
  const orgId = await resolveOrgId(req)
  const config = await getCallRoutingConfig(orgId)

  // Usar VAPI key do org (Story 12.8) ou fallback env
  const vapiKey = config?.integrations?.vapi?.apiKey || process.env.VAPI_API_KEY
  if (!vapiKey) return NextResponse.json({ error: 'VAPI nao configurado' }, { status: 400 })

  const response = await fetch('https://api.vapi.ai/voice', {
    headers: { Authorization: `Bearer ${vapiKey}` },
  })
  const voices = await response.json()

  // Filtrar por idioma se solicitado
  const lang = req.nextUrl.searchParams.get('lang') || 'pt'
  const filtered = voices.filter(v => !lang || v.language?.includes(lang))

  return NextResponse.json(filtered)
}
```

**Override de voz no `makeVapiCall()`:**

// Em makeVapiCall(), adicionar ao assistantOverrides:
