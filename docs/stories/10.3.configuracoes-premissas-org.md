# Story 10.3 — Configuracoes e Premissas por Organizacao

**Status:** Done
**Epic:** 10 — Revisao de Propostas Comerciais
**Priority:** High
**Complexity:** Medium (5-8 points)

---

## Story

Como **admin de uma organizacao**, eu quero configurar as premissas de desconto (desenvolvimento, especificacao, testes) e parametros de proposta (minimo de produtos para desconto, condicoes de pagamento padrao), para que cada organizacao tenha seus proprios valores em vez de depender de constantes hardcoded.

---

## Problem

Os valores de desconto (20% desenvolvimento, 70% especificacao, 50% testes) e o threshold de produtos para desconto progressivo (>2) estavam hardcoded em ambas as paginas de proposta (nova e edicao). Organizacoes diferentes nao podiam personalizar essas premissas.

---

## Acceptance Criteria

- [x] AC1: Existe tipo ProposalConfig com campos configuracaveis de premissas
- [x] AC2: Hook useProposalConfig carrega config da org em tempo real (onSnapshot)
- [x] AC3: Tab Configuracoes no admin permite editar: desconto de desenvolvimento, especificacao, teste, minimo de produtos, condicoes de pagamento
- [x] AC4: Config salva em `organizations/{orgId}/settings/proposalConfig`
- [x] AC5: Pagina nova/proposta usa orgConfig em vez de valores hardcoded
- [x] AC6: Pagina edicao/proposta usa orgConfig em vez de valores hardcoded
- [x] AC7: Premissas sincronizam uma vez ao carregar (flag premissasInitialized)
- [x] AC8: Default config usado quando org nao tem configuracao salva
- [x] AC9: TypeScript compila sem erros
- [x] AC10: ESLint passa sem warnings
