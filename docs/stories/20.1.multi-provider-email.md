## Story

Como **administrador do CRM**, eu quero escolher entre diferentes provedores de email (Resend, SendGrid) e configurar credenciais, para que a organizacao tenha flexibilidade e redundancia no envio de emails.

---

## Problem

O sistema esta acoplado a um unico provedor de email. Se o provedor tiver problemas ou limites, nao ha fallback. Diferentes organizacoes podem preferir provedores diferentes (custo, features, deliverability). Falta uma abstraction layer para provedores de email.

## Dev Notes

**Pattern:** Strategy pattern com factory. Cada provider implementa a mesma interface.

```typescript
interface EmailProvider {
  send(to: string, subject: string, html: string): Promise<EmailResult>
  sendBulk(emails: BulkEmail[]): Promise<BulkEmailResult>
}
```

**Credenciais:** Armazenar API keys na collection `organizationSecrets` (encrypted) ou usar env vars por ora.

**Fallback:** Try/catch no provider primario → se falhar, tentar secundario.
