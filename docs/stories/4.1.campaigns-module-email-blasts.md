## Story

**Como** usuário do CRM com plano Pro,
**Quero** criar campanhas de email segmentando contatos por múltiplos filtros, disparar emails em massa, agendar envios futuros e recorrentes, e ter histórico completo de campanhas e envios,
**Para que** eu possa executar ações de marketing e nutrição de leads de forma organizada, rastreável e escalável.

## Acceptance Criteria

### Bloco A — Página de Campanhas e Listagem

#### AC-1: Ativação da rota `/campanhas` no sidebar
**Given** o módulo de campanhas está implementado
**When** o usuário com plano Pro acessa o CRM
**Then** o item "Campanhas" no sidebar está ativo (sem badge "Em breve")
**And** ao clicar, navega para `/campanhas`
**And** se o plano NÃO é Pro, exibe `PlanGate` com `UpgradePrompt`

#### AC-2: Listagem de campanhas
**Given** o usuário acessa `/campanhas`
**When** a página carrega
**Then** exibe uma tabela/grid com campanhas da organização, contendo:
- Nome da campanha
- Status (Rascunho | Agendada | Enviando | Concluída | Falha parcial)
- Total de destinatários
- Emails enviados / falhos
- Data de criação
- Data de envio (ou próximo envio se recorrente)
- Tipo (Imediata | Agendada | Recorrente)
**And** ordenadas por data de criação (mais recente primeiro)
**And** com paginação (20 por página)

#### AC-3: Filtros na listagem de campanhas
**Given** o usuário está na listagem de campanhas
**When** ele utiliza os filtros
**Then** pode filtrar por: status, tipo (imediata/agendada/recorrente), período de criação
**And** busca por nome da campanha

### Bloco B — Criação de Campanha e Segmentação

#### AC-4: Fluxo de criação em etapas (wizard)
**Given** o usuário clica em "Nova Campanha"
**When** o wizard de criação inicia
**Then** segue 3 etapas sequenciais:
1. **Segmentação** — selecionar destinatários
2. **Composição** — escrever o email
3. **Revisão e Envio** — confirmar e disparar/agendar

#### AC-5: Segmentação de contatos com filtros avançados
**Given** o usuário está na etapa 1 (Segmentação)
**When** ele aplica filtros
**Then** os filtros disponíveis são:
- **Funil e Etapa**: filtrar por funil específico e/ou etapa
- **Status do contato**: Lead, Lead-qualificado, Ativo, Inativo, Inatividade Longa
- **Origem do lead** (`leadSource`): todos os valores existentes (Google, Instagram, Indicação, etc.)
- **Tipo do lead** (`leadType`): Inbound, Outbound
- **Segmento/Indústria** (`industry`): campo texto com autocomplete
- **Empresa** (`company`): campo texto com busca
- **Porte da empresa** (`porte_empresa`): filtro de seleção
- **Localização**: Estado (`estado`) e Município (`municipio`)
- **Tipo de empresa** (`tipo`): filtro de seleção
- **Natureza jurídica** (`natureza_juridica`): filtro de seleção
- **Capital social** (`capital_social`): range (de/até)
- **Tempo desde último contato**: range em dias (ex: "sem contato há mais de 30 dias")
- **Tempo desde última atividade** (`lastFollowUpAt`): range em dias
- **Responsável** (`assignedTo`): filtro por membro (depende Story 2.2)
- **Data de criação** (`createdAt`): range de datas
- **Tem email**: checkbox (filtrar contatos COM email preenchido)
**And** os filtros são combinados com lógica AND
**And** ao aplicar filtros, exibe a contagem de contatos encontrados em tempo real

#### AC-6: Seleção massiva de contatos
**Given** os filtros foram aplicados e os contatos são exibidos
**When** o usuário visualiza a lista de resultados
**Then** vê uma tabela paginada com: checkbox, nome, empresa, email, telefone, etapa, responsável
**And** há checkbox "Selecionar todos" que marca TODOS os contatos filtrados (não só a página visível)
**And** pode desmarcar contatos individuais manualmente
**And** um contador exibe "X de Y contatos selecionados"
**And** contatos SEM email são exibidos com indicador visual (ícone de alerta) e NÃO são selecionáveis

#### AC-7: Salvar segmento como filtro reutilizável
**Given** o usuário configurou uma combinação de filtros
**When** ele clica em "Salvar segmento"
**Then** pode dar um nome ao segmento (ex: "Leads inativos SP")
**And** o segmento é salvo e pode ser reutilizado em campanhas futuras
**And** segmentos salvos aparecem como opção rápida no topo da área de filtros

### Bloco C — Composição do Email

#### AC-8: Editor de email com Tiptap
**Given** o usuário está na etapa 2 (Composição)
**When** ele compõe o email
**Then** vê os campos:
- **Assunto** (input text, obrigatório, max 150 chars)
- **Corpo do email** (editor Tiptap com toolbar completa)
**And** o editor Tiptap oferece:
- Formatação: negrito, itálico, sublinhado, riscado
- Cores de texto e destaque
- Títulos (H1, H2, H3)
- Listas (ordenadas e não-ordenadas)
- Links (inserir/editar URL)
- Imagens (upload ou URL)
- Alinhamento (esquerda, centro, direita)
- Placeholder text

#### AC-9: Variáveis dinâmicas no email
**Given** o usuário está escrevendo o corpo do email
**When** ele clica no botão "Inserir variável" (ou digita `{{`)
**Then** vê um dropdown com variáveis disponíveis:
- `{{nome}}` — nome do contato
- `{{empresa}}` — empresa do contato
- `{{email}}` — email do contato
- `{{responsavel}}` — nome do responsável
**And** as variáveis são substituídas individualmente para cada destinatário no momento do envio

#### AC-10: Preview do email
**Given** o email está composto com variáveis
**When** o usuário clica em "Preview"
**Then** vê uma prévia do email renderizado com dados de um contato de exemplo (primeiro da seleção)
**And** pode navegar entre "Preview" e "Edição" sem perder conteúdo

### Bloco D — Envio e Agendamento

#### AC-11: Envio imediato
**Given** o usuário está na etapa 3 (Revisão e Envio)
**When** ele seleciona "Enviar agora"
**Then** vê um resumo: total de destinatários, assunto, preview do corpo
**And** ao confirmar, a campanha é salva com status "Enviando"
**And** os emails são disparados em lotes (batch) para evitar rate limit do Gmail
**And** o progresso é exibido: "Enviando... 45/120"

#### AC-12: Agendamento para data futura
**Given** o usuário está na etapa 3
**When** ele seleciona "Agendar envio"
**Then** vê um date-time picker para selecionar data e hora do envio
**And** a data deve ser no futuro (mínimo 15 minutos)
**And** ao confirmar, a campanha é salva com status "Agendada"
**And** o sistema exibe: "Campanha agendada para DD/MM/YYYY às HH:MM"

#### AC-13: Envio recorrente (periodicidade)
**Given** o usuário está na etapa 3
**When** ele seleciona "Envio recorrente"
**Then** pode configurar:
- **Frequência**: Diário, Semanal, Quinzenal, Mensal
- **Dia da semana** (se semanal/quinzenal): segunda a sexta
- **Dia do mês** (se mensal): 1-28
- **Hora do envio**: time picker
- **Data de início**: date picker
- **Data de término** (opcional): date picker ou "Sem término"
**And** a segmentação é re-executada a cada envio (novos contatos que se encaixam nos filtros são incluídos)
**And** a campanha é salva com status "Agendada" e tipo "Recorrente"

#### AC-14: Processamento em lotes (batch) com rate limiting
**Given** uma campanha está sendo enviada
**When** o sistema processa os envios
**Then** envia em lotes de no máximo 20 emails por vez
**And** aguarda 2 segundos entre cada lote (respeitar limites Gmail)
**And** se um email falha, registra o erro e continua com os próximos
**And** ao final, atualiza o status: "Concluída" (100% sucesso) ou "Falha parcial" (com count de falhos)

### Bloco E — Histórico e Logs

#### AC-15: Detalhes da campanha enviada
**Given** uma campanha foi enviada (total ou parcialmente)
**When** o usuário clica na campanha na listagem
**Then** vê a página de detalhes com:
- Resumo: nome, assunto, data envio, total enviados/falhos
- Corpo do email renderizado (como foi enviado)
- Lista de destinatários com status individual: Enviado | Falhou | Pendente
- Filtros usados na segmentação (para referência)
**And** pode re-enviar para contatos que falharam ("Reenviar falhos")

#### AC-16: Log no contato individual
**Given** um email de campanha foi enviado para um contato
**When** o envio é processado
**Then** um log é criado em `clients/{clientId}/logs` com:
- `action`: "campaign_email_sent"
- `message`: "Email enviado — Campanha: [Nome da campanha]"
- `metadata`: `{ campaignId, campaignName, subject, status: 'sent' | 'failed', error? }`
- `author`: "Sistema (Campanha automática)"
- `createdAt`: timestamp do envio
**And** o log é visível no histórico de atividades do contato

#### AC-17: Dashboard resumo de campanhas
**Given** o usuário acessa `/campanhas`
**When** a página carrega
**Then** vê no topo métricas resumidas:
- Total de campanhas enviadas (mês atual)
- Total de emails enviados (mês atual)
- Taxa de falha média
- Próximo envio agendado (se houver)

## Modelo de Dados — Novas Collections

### Campaign (em `organizations/{orgId}/campaigns`)
```typescript
interface Campaign {
  id: string
  orgId: string
  name: string
  subject: string
  body: string                    // HTML do Tiptap
  bodyPlainText: string           // Versão plain text
  status: 'draft' | 'scheduled' | 'sending' | 'completed' | 'partial_failure' | 'cancelled'
  type: 'immediate' | 'scheduled' | 'recurring'

  // Segmentação
  filters: CampaignFilters        // Filtros usados
  savedSegmentId?: string         // Se usou segmento salvo
  totalRecipients: number         // Total na hora do envio

  // Agendamento
  scheduledAt?: string            // ISO date para envio agendado
  recurrence?: CampaignRecurrence // Config de recorrência

  // Resultados
  sentCount: number
  failedCount: number
  lastSentAt?: string

  // Metadata
  createdBy: string               // memberId
  createdByName: string
  createdAt: string
  updatedAt: string
}

interface CampaignFilters {
  funnelId?: string
  stageIds?: string[]
  status?: string[]
  leadSource?: string[]
  leadType?: string[]
  industry?: string
  company?: string
  porteEmpresa?: string[]
  estado?: string[]
  municipio?: string[]
  tipo?: string[]
  naturezaJuridica?: string[]
  capitalSocialMin?: number
  capitalSocialMax?: number
  daysSinceLastContact?: number   // "sem contato há mais de X dias"
  daysSinceLastActivity?: number
  assignedTo?: string             // filtro por responsável
  createdAfter?: string
  createdBefore?: string
  hasEmail: boolean               // true = somente com email
}

interface CampaignRecurrence {
  frequency: 'daily' | 'weekly' | 'biweekly' | 'monthly'
  dayOfWeek?: number              // 0=domingo ... 6=sábado
  dayOfMonth?: number             // 1-28
  timeOfDay: string               // "HH:MM" formato 24h
  startDate: string               // ISO date
  endDate?: string                // ISO date ou null (sem término)
  nextRunAt: string               // Próximo envio calculado
}
```

## Dev Notes

### Referência de Arquitetura
- Consultar `docs/architecture/system-architecture.md` para visão geral
- Email está documentado como serviço via Gmail/Nodemailer
- Rate limit do Gmail: ~500 emails/dia para contas regulares, ~2000/dia para Google Workspace

### Rate Limiting do Gmail — IMPORTANTE
Gmail tem limites rigorosos:
- **Conta pessoal**: ~500 emails/dia
- **Google Workspace**: ~2.000 emails/dia
- **Recomendação**: Lotes de 20 emails com 2s entre lotes (600 emails/hora)
- **Futuro**: Migrar para SendGrid/Resend/Amazon SES para volume maior

### Estratégia de Agendamento
Duas abordagens para processar campanhas agendadas:

**Opção A — API Route + Cron externo:**
```
cron job (ex: cron-job.org, Vercel Cron) → GET /api/campaigns/process-scheduled
- Executa a cada 5 minutos
- Busca campanhas pendentes, dispara envio
```

**Opção B — Firebase Cloud Functions (se disponível):**
```
Cloud Function com Pub/Sub scheduler → processa campanhas
- Mais confiável, menos dependência externa
```

**Recomendação:** Opção A é mais simples e não requer setup de Cloud Functions.
