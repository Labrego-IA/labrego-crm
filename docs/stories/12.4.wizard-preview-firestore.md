# Story 12.4 — Preview ao Vivo do Prompt + Persistencia Firestore

**Status:** Ready for Review
**Epic:** 12 — Wizard Gamificado de Configuracao do Agente de Voz
**Priority:** High
**Complexity:** Medium (5 points)
**Depends On:** Story 12.2, Story 12.3

---

## Story

Como **usuario que configurou o agente pelo wizard**, eu quero ver em tempo real o prompt completo que esta sendo gerado a partir das minhas respostas, e ter a certeza de que tudo e salvo e usado pelo agente nas ligacoes, para ter confianca de que o agente vai se comportar como eu configurei.

---

## Problem

Mesmo com o wizard preenchido, o usuario nao tem visibilidade do prompt final gerado. Ele nao sabe se as respostas estao sendo "traduzidas" corretamente. Alem disso, o prompt montado precisa ser persistido no Firestore e usado pelo `buildSystemPrompt()` nas ligacoes reais.

## Dev Notes

**Fluxo de salvamento:**
```typescript
const handleSaveAgent = async () => {
  const prompt = assemblePromptFromWizard(wizardAnswers)
  const strength = calculateAgentStrength(wizardAnswers)

  const updatedKnowledge = {
    ...agentKnowledge,
    systemPrompt: prompt,  // Prompt montado pelo wizard
    wizardAnswers: { ...wizardAnswers, strengthScore: strength },
    // Manter campos individuais sincronizados para backward compat
    agentName: wizardAnswers.agentName,
    agentRole: wizardAnswers.agentRole,
    companyName: wizardAnswers.companyName,
    // ...etc
  }

  await setDoc(configRef, { agentKnowledge: updatedKnowledge }, { merge: true })
}
```

**Preview layout:**

```
┌─ Wizard (70%) ──────────────┬─ Preview (30%) ──────────┐
│                              │ # LEO — SDR Senior       │

│                              │ ## IDENTIDADE             │

│  ┌───────────────────┐       │                           │
│  │ Ola {{contactName}}│       │ ## PRINCIPIOS...          │
│  └───────────────────┘       │ ...                       │

└──────────────────────────────┴───────────────────────────┘
```

**Compatibilidade com `buildSystemPrompt()`:**
A funcao `buildSystemPrompt()` em `callRouting.ts` le `agentKnowledge.systemPrompt`. Como o wizard salva o prompt montado nesse campo, nao precisa de mudancas na hora de fazer a ligacao.
