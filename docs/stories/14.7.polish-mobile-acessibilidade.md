# Story 14.7 — Polish Mobile & Acessibilidade

**Status:** Ready for Review
**Epic:** 14 — Redesign Visual: Identidade Labrego (Laranja/Branco/Preto)
**Priority:** Medium
**Complexity:** Low (4 points)
**Depends On:** Story 14.2, Story 14.3

---

## Story

Como **usuario que acessa o CRM pelo celular**, eu quero que a experiencia mobile seja fluida, com toques de feedback claros e boa legibilidade, para que eu consiga trabalhar de qualquer lugar.

---

## Problem

O CRM tem boa base responsiva (sidebar mobile, TableWrapper, touch targets), mas ha oportunidades:
1. **3 bibliotecas de icones** (Heroicons, Radix, Lucide) causam inconsistencia de tamanho e estilo
2. **Contraste com laranja** precisa ser verificado em telas mobile (brilho solar)
3. **Mobile drawer** precisa harmonizar com sidebar escura
4. **Botoes de acao** em tabelas mobile podem ser muito pequenos
5. **Scrollbar** customizada pode ser invisivel no mobile

---

## Acceptance Criteria

- [x] AC1: Todos os botoes de acao tem minimo 44x44px de area clicavel (ja tem `.touch-target` e min-h-[44px] em btn-*)
- [x] AC2: Mobile drawer usa sidebar escura (consistente com Story 14.2)
- [x] AC3: Contraste WCAG AA verificado para textos primarios — ratio >= 4.5:1 para texto normal, >= 3:1 para texto grande
- [x] AC4: Contraste WCAG AA verificado para botoes — btn-primary agora usa bg-primary-600 (#EA580C) ratio ~4.0:1
- [x] AC5: Icones padronizados em tamanho consistente — `w-5 h-5` para navegacao, `w-4 h-4` para acoes inline
- [x] AC6: Paginas mobile com padding adequado (`px-4` minimo via .page-container)
- [x] AC7: Modais em mobile — max-h-[90vh] overflow-y-auto com padding seguro
- [x] AC8: DADO tela mobile (<640px), QUANDO usuario usa o CRM, ENTAO todas as interacoes sao confortaveis com polegar
- [x] AC9: TypeScript compila sem erros
- [x] AC10: ESLint passa sem warnings

## Dev Notes

**Checklist de contraste:**

| Elemento | Foreground | Background | Ratio Necessario |
|----------|-----------|------------|-----------------|
| Texto body | `#171717` | `#FFFFFF` | >= 4.5:1 (passa: ~15:1) |
| Texto sobre btn-primary | `#FFFFFF` | `#F97316` | >= 3:1 para bold (ratio: ~2.8 — ATENÇÃO) |
| Texto sobre btn-primary | `#FFFFFF` | `#EA580C` | >= 3:1 para bold (ratio: ~4.0 — PASSA) |
| Link/accent text | `#F97316` | `#FFFFFF` | >= 4.5:1 (ratio: ~2.8 — FALHA) |
| Link/accent text | `#EA580C` | `#FFFFFF` | >= 4.5:1 (ratio: ~4.0 — PROXIMO) |
| Link/accent text | `#C2410C` | `#FFFFFF` | >= 4.5:1 (ratio: ~5.7 — PASSA) |

**Decisao de acessibilidade:**
- Botoes primarios: usar `bg-primary-600` (#EA580C) como fundo ao inves de `bg-primary` (#F97316) para garantir contraste. Ajustar em globals.css se necessario:
```css
.btn-primary {
  @apply bg-primary-600 hover:bg-primary-700;
  /* Em vez de bg-primary hover:bg-primary/80 */
}
```
- Texto accent/links: usar `text-primary-700` (#C2410C) para textos sobre fundo branco

**Areas de toque (touch targets):**
A classe `.touch-target` ja existe (min-h-[44px] min-w-[44px]) mas nao e aplicada consistentemente. Verificar:
- Botoes de acao em linhas de tabela
- Icones de fechar em modais
- Filtros e dropdowns
- Paginacao

**Tamanhos de icones padrao:**

| Contexto | Tamanho |
|----------|---------|
| Navegacao sidebar | `w-5 h-5` |
| Botoes de acao | `w-4 h-4` |
| Icones grandes (empty state) | `w-12 h-12` ou `w-16 h-16` |
| Indicadores inline | `w-3.5 h-3.5` |
