# Story 11.2 — Sinalizacao de Propostas no Card do Funil

**Status:** Ready
**Epic:** 11 — Enriquecimento do Card e Painel de Contato no Funil
**Priority:** High
**Complexity:** Medium (5 points)
**Depends On:** None

---

## Story

Como **usuario que visualiza o funil**, eu quero ver no card do contato uma sinalizacao visual de que existe proposta vinculada, incluindo o valor e status, para que eu saiba rapidamente quais contatos ja tem proposta sem precisar abrir o detalhe.

---

## Problem

Atualmente o card do contato no kanban mostra informacoes como nome, empresa, tipo de lead, dias no estagio, etc — mas nao mostra nenhuma informacao sobre propostas. O usuario precisa clicar no contato e navegar ate a aba de propostas para saber se ha proposta e qual o valor. Isso torna o processo de triagem lento.

---

## Acceptance Criteria

- [ ] AC1: Carregar dados de propostas agrupados por `clientId` ao montar o funil
- [ ] AC2: Card do contato exibe badge de proposta quando ha pelo menos uma proposta vinculada
- [ ] AC3: Badge mostra o valor total da proposta mais recente (formatado em BRL)
- [ ] AC4: Badge mostra o status da proposta mais recente com cor indicativa
- [ ] AC5: Cores de status: verde=Aprovada, amarelo=Pendente, azul=Em analise, vermelho=Recusada, cinza=Expirada/Cancelada
- [ ] AC6: Se ha multiplas propostas, mostrar indicador de quantidade (ex: "+2")
- [ ] AC7: Se nao ha proposta, nenhum badge e exibido (card limpo)
- [ ] AC8: Badge nao deve impactar performance do kanban (query otimizada)
- [ ] AC9: TypeScript compila sem erros
- [ ] AC10: ESLint passa sem warnings

## Dev Notes

**Query sugerida (Firestore):**
```typescript
// No useEffect de carregamento do funil
const proposalsSnap = await getDocs(
  query(collection(db, 'proposals'), where('orgId', '==', orgId))
)
const proposalsByClient: Record<string, { total: number; status: string; count: number }> = {}
proposalsSnap.docs.forEach(d => {
  const data = d.data()
  const cid = data.clientId
  if (!proposalsByClient[cid]) {
    proposalsByClient[cid] = { total: data.total || 0, status: data.status || 'Pendente', count: 1 }
  } else {
    proposalsByClient[cid].count++
    // Manter a mais recente (ou maior valor — definir criterio)
  }
})
```

**Badge visual sugerido no card:**
```
┌────────────────────────┐
│ 📄 R$ 15.000 🟡       │  ← badge no topo do card
│ João Silva             │

```

**Performance:**
- Carregar propostas uma unica vez com listener ou query
- Nao fazer query individual por contato
- Mapa de lookup `clientId → proposalSummary` e O(1) por card
