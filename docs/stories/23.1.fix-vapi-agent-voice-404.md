# Story 23.1 — Fix VAPI Agent Voice 404 Error

**Status:** Ready
**Epic:** 23 — Bug Fixes
**Points:** 2
**Priority:** Critical (feature broken)
**Dependencies:** Nenhuma

---

## Story

Como usuário do CRM, quero que a funcionalidade de voz do agente (VAPI) funcione corretamente, pois atualmente ao tentar usar o agente de voz está retornando erro 404, impossibilitando o uso de ligações automáticas.

---

## Acceptance Criteria

### AC1: Diagnosticar causa do 404
- **Given** o erro 404 ao usar o agente de voz
- **When** o desenvolvedor investiga
- **Then** identifica a causa raiz (possíveis causas abaixo)
- **And** documenta a causa no Debug Log

### AC2: Corrigir o erro
- **Given** a causa identificada
- **When** a correção é aplicada
- **Then** a funcionalidade de voz do agente volta a funcionar sem erro 404
- **And** as chamadas são iniciadas corretamente via VAPI

### AC3: Verificar integração completa
- **Given** o fix aplicado
- **When** o fluxo completo é testado
- **Then** a chamada é iniciada (sem 404)
- **And** o webhook de fim de chamada processa corretamente
- **And** o status da chamada é atualizado no CRM

### OUT of Scope

## Dev Notes

### Possíveis causas do 404
1. **VAPI_API_KEY inválida ou expirada** — verificar em `.env.local`
2. **Assistant ID não encontrado no VAPI** — o assistantId pode ter sido deletado/alterado no dashboard VAPI
3. **Rota de API incorreta** — alguma rota em `src/app/api/vapi/` pode ter mudado
4. **Phone number não configurado** — o número de telefone VAPI pode não estar associado ao assistente
5. **URL de webhook incorreta** — se o VAPI webhook aponta para URL errada

### Arquivos VAPI existentes
- `src/app/api/vapi/webhook/route.ts` — webhook principal
- `src/app/api/vapi/poll-call/route.ts` — polling de status
- `src/app/api/vapi/voices/route.ts` — vozes disponíveis
- `src/app/api/vapi/tools/available-slots/route.ts` — slots de agenda
- `src/app/api/vapi/tools/schedule-meeting/route.ts` — agendar reunião

### Onde a chamada é iniciada
Buscar no funil page (`src/app/funil/[funnelId]/page.tsx`) ou na página de ligações (`src/app/ligacoes/`) por:
- Chamada para VAPI API (fetch para `https://api.vapi.ai/`)
- Referência a `VAPI_API_KEY` ou `NEXT_PUBLIC_VAPI_API_KEY`
- Componente de iniciar chamada

### Debug steps para o dev
1. Verificar se `VAPI_API_KEY` está definida em `.env.local`
2. Verificar se o assistantId usado no código existe no dashboard VAPI
3. Testar manualmente: `curl -H "Authorization: Bearer $VAPI_API_KEY" https://api.vapi.ai/assistant`
4. Verificar logs do console do browser para a request 404 (qual URL exata retorna 404)
5. Verificar se a URL base da VAPI API está correta
